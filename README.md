



# Statistical and biological analysis of CoRegTor  

This repository contains code for analysis of CoRegTor, a computational tool that predicts a potential set of genes that coordinate the transcription regulation of a target gene. 

The tool itself is available in a separate repository and can be installed as a python package. See [https://github.com/shubhvjain/coregtor](https://github.com/shubhvjain/coregtor/) for more details on how to use CoRegTor. 

Here we focus on various downstream analysis performed on the results generated by the tool to demonstrate its validity using statistical and biological techniques. We also compare the performance of CoRegTor with other tools with the same purpose. 

## Analysis overview 

We define the workflows for multiple experiment types. Experiments follow a similar workflow thought steps varies internally. 

Experiment workflow:
- Initialization 
- Experiment Run 
- Result generation 
- Analysis of results

We later describe each step in details for individual experiment types. 

Each experiment is described in a self contained json file.These files are located in the `experiments` folder. The name of the file is also the name of the experiment. Since running these experiments may generates huge amounts of intermediate files, these are not included here. But the results generated from the intermediate run files are included. The results files are located in the `outputs` folder within a folder with the same experiment name.

## Setting up the environment

Before running an experiment, it is important to make sure you are using the correct environment because different types of containers are run in different environment. 

This is controlled by the `tfit_mode` variable. We provide 2 modes:
- `local`: use this mode to run on a system where docker is available. This uses the `.env.local` file 
- `hpc`: use this mode to run on a system where apptainer is available. Note that this is specifically configured to run on the [HPC](https://hpc.fau.de/) resources provided by the Erlangen National High Performance Computing Center (NHR@FAU) of the Friedrich-Alexander-Universität Erlangen-Nürnberg (FAU).

# Instructions for setup on HPC@FAU



## Poetry Installation on FAU Woody HPC
Woody's default Python (3.9) is too old for Poetry 2.3+, so you need to load the 3.12 module first.

### One-time setup
```bash
module load python/3.12-conda
export POETRY_HOME=$WORK/software/poetry
curl -sSL https://install.python-poetry.org | python3 -
poetry config virtualenvs.path $WORK/software/venvs
```

### Add to ~/.bashrc (persistent)
```bash
module load python/3.12-conda
export POETRY_HOME=$WORK/software/poetry
export PATH="$POETRY_HOME/bin:$PATH"
```

### Notes
- Always install to `$WORK`, not `$HOME` — home quota is very limited on Woody
- `virtualenvs.path` is also set to `$WORK` for the same reason
